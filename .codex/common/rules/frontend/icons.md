# Frontend Icon Rules

Use these rules when choosing and applying icons.


## Icon System

Use Font Awesome as the default icon system when the active stack supports it.


Use the package set defined by the active stack's `decisions.md`.


## Button Icons

Buttons should include an icon that matches the button's action or meaning when a recognizable icon exists.


Common mappings:

- Add or create: `faPlus`
- Save: `faFloppyDisk`
- Edit: `faPenToSquare`
- Delete: `faTrash`
- Search: `faMagnifyingGlass`
- Filter: `faFilter`
- Download: `faDownload`
- Upload: `faUpload`
- Settings: `faGear`
- Close or cancel: `faXmark`
- Back or previous: `faArrowLeft`
- Next or forward: `faArrowRight`


## Anti-Patterns

- Do not use decorative icons that do not clarify the action.
- Do not use multiple icon systems without a clear compatibility reason.
- Add labels or tooltips when an icon is not obvious.
